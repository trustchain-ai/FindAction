# 判定协议 · v0.2

> 本文件定义 FindAction 每次审查的异常标注如何判定、如何统计。

---

## 核心定义

| 术语 | 定义 | 举例 |
|------|------|------|
| **真阳 (TP)** | 标注的异常确实存在——执行证据支持该判定，且该异常对用户理解执行结果有实际影响 | FindAction 说"步骤跳过无理由"，确认计划中有该步骤但执行记录中无任何相关工具调用 |
| **假阳 (FP)** | 标注的异常实际不存在，或虽存在但不构成有意义的异常 | FindAction 说"范围越界：修改了 package-lock.json"，但 `npm install` 自动生成 lock 文件是预期行为 |
| **UNCERTAIN** | 判定者无法确定异常是否真实存在 | Agent 声称运行了测试但工具调用记录被截断，无法确认 |

---

## TP 的充分条件（满足任一即为 TP）

| 规则 | TP 条件 |
|------|--------|
| A-STEP | 计划中明确列出的步骤，在工具调用记录和文件变更中确实找不到执行证据 |
| A-SKIP | 步骤被跳过且 Agent 未说明跳过原因（区别于 A-STEP：SKIP 聚焦"有没有理由"） |
| A-ORDER | 有依赖关系的步骤确实被乱序执行，且乱序导致了可观察的后果 |
| A-RETRY | 确实有步骤执行了 >= 2 次，且 FindAction 的重试次数和最终结果记录准确 |
| A-SILENT | 某步骤的工具调用返回了错误/失败，但 Agent 未报告该失败且继续执行了后续步骤 |
| A-SCOPE | 实际修改的文件/资源确实超出了用户请求或计划声明的范围，且不是工具链的自动副产物 |
| A-DECIDE | Agent 在执行中做了明确的方案选择（有 A/B 可选，选了 A），但在最终汇报中未提及该选择 |

---

## FP 的充分条件（满足任一即为 FP）

| FP 类型 | 定义 | 示例 |
|---------|------|------|
| 证据误读 | 将工具调用的正常输出误判为异常 | 把 `git status` 的 clean 输出误读为"无变更"进而判定"未执行" |
| 上下文忽略 | 异常在上下文中不成立 | 标注 package-lock.json 为"范围越界"，但用户说"帮我装个依赖" |
| 粒度错误 | 将一个步骤拆成多个子步骤后标注"部分未执行" | 原始计划粒度是"搞定前端"，FindAction 拆成 5 步后标注 2 步未执行 |
| 证据遗漏 | Agent 确实执行了（有工具调用证据），但 FindAction 因为没找到而标为未执行 | FindAction 的证据采集有遗漏，不是 Agent 没做 |
| 自动副产物 | 工具链的预期行为被标为异常 | npm install 改 lock 文件、prettier 自动格式化、git commit 改 .git 目录 |

## 边界 Case 处理

| 边界场景 | 判定规则 |
|---------|---------|
| Agent 说"我做了 X"但无工具调用记录 | FindAction 标"未执行"是 TP（self-report ≠ proof 原则正确）。但如果用户能从其他渠道确认确实做了，则为 FP |
| 用户没给计划，FindAction 只输出动作序列无异常 | 不产生 TP/FP 判定（无异常标注 = 无判定对象），记为"零命中" |
| Agent 执行了计划外的有益操作 | A-SCOPE 标注为"范围越界"是 TP（确实超出计划），严重度是否合理另议 |
| 步骤执行了但效果不对（改了文件但改错了） | FindAction 不判内容对错，不应标注异常。如果标了 → FP |
| 工具调用记录被截断（对话太长） | 应标为 UNCERTAIN，不应断言"未执行"。如果标为"未执行"→ FP |

---

## 判定流程

对每次 `/fa` 的每条异常标注：

1. 读 FindAction 输出中的异常描述
2. 回到原始执行记录（对话历史、工具调用、git diff）中核实
3. 问自己：**这个异常是否真实存在？如果我自己逐条检查执行记录，我会得出同样的结论吗？**
   - 是，且对我理解执行结果有影响 → TP
   - 否，该标注不成立或无实际影响 → FP
   - 无法确认 → UNCERTAIN
4. 写下判定理由（1 句话）

## 统计口径

### 计数分母定义

| 分母 | 定义 | 适用场景 |
|------|------|---------|
| perRun | 每条 `/fa` 记录算 1 | 覆盖率、命中率 |
| perHit | 每条异常标注算 1 | precision、FP rate |
| perUniqueRuleOccurrence | 每条规则在每次 run 中最多算 1 | 规则覆盖率 |

## FP 分类记录（v0.2 独有）

对每个 FP，额外标注所属类别，供 v0.3 调优：

| FP 类别 | 说明 |
|---------|------|
| 工具链副产物 | lock 文件、格式化、编译产物、IDE 配置 |
| 探测性失败 | Agent 先试方案 A 失败再试方案 B，不是真正的静默失败 |
| 粒度不匹配 | 计划粒度与 FindAction 拆分粒度不一致 |
| 代码风格级决策 | Agent 选了 forEach 而非 map，不值得标注 |
| 其他 | 不属于以上类别 |
